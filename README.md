# Анализ транзакций контрагентов

## Описание
Тестовое задание: анализ финансовых транзакций, очистка данных, SQL-аналитика и ML-классификация описаний по категориям ОКЭД.

## Структура репозитория
- notebooks/analysis.ipynb — основной ноутбук
- sql/queries.sql — SQL запросы с window-функциями
- src/model.pkl — обученная модель
- src/vectorizer.pkl — TF-IDF векторайзер

## Данные
- transactions.csv — 80,800 транзакций за период 2024-01 — 2026-04
- categories.json — 20 категорий ОКЭД с ключевыми словами
- gold_set.csv — 200 размеченных примеров для оценки модели

## Часть 1 — EDA и очистка данных

### Выявленные аномалии
| Проблема | Количество |
|---|---|
| Даты в 4 разных форматах | 34,566 строк (42.8%) |
| Отрицательные суммы | 384 транзакции |
| Пропуски в описании | 847 строк |
| Невалидные БИН/ИИН | 1,190 sender + 1,126 receiver |
| Дубликаты | 800 строк |

### После очистки
- Все даты приведены к единому формату
- Удалено 800 дубликатов
- Итого строк: 80,000

## Часть 2 — SQL
Window-функции: RANK, SUM OVER, AVG OVER PARTITION BY, ROW_NUMBER

## Часть 3 — ML классификация

### Подход
1. Разметка по ключевым словам из categories.json — 75,747 примеров
2. TF-IDF (8000 признаков, ngram 1-3) + LogisticRegression
3. Rule-based постобработка для сложных случаев

### Результаты
| Метрика | Значение |
|---|---|
| Accuracy на test split | 99.97% |
| Accuracy на gold_set | 100% (200/200) |
